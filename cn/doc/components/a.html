a

<a> (v0.5+)

<a> 组件定义了指向某个页面的一个超链接. 此组件的作用和用法与HTML5中的 <a> 非常类似，区别在于 Weex 的 <a> 组件不能直接在里面添加文本（字符串），如果要展示文本，应该添加 <text> 组件。

一个简单例子：

体验一下

<template>
  <div class="wrapper">
    <a href="http://dotwe.org/raw/dist/a468998152ee680413588c38bd61c29e.js">
      <text>click</text>
    </a>
  </div>
</template>

<style>
.wrapper {
  text-align: center;
}
</style>

子组件

此组件支持除了自己外的所有 Weex 组件作为子组件。

特性

  • href： 定义了超链接的 URL。

样式

<a> 支持所有通用样式。

  • 盒模型
  • flexbox 布局
  • position
  • opacity
  • background-color

查看 组件通用样式。

事件

<a> 支持所有通用事件。

  • click 注意：我们不能保证 click 事件和 href 跳转的执行顺序。建议不要使用 click 事件来处理 href 跳转前的逻辑处理。
  • longpress
  • appear
  • disappear

查看 通用事件。

约束

  1. 不能直接在 <a> 中添加文本。 错误示例，“click” 无法被正常渲染。体验一下

    <template>
 <div class="wrapper">
   <a href="http://dotwe.org/raw/dist/a468998152ee680413588c38bd61c29e.js">
     click
   </a>
 </div>
</template>

<style>
.wrapper {
 text-align: center;
}
</style>

  2. 请不要为 <a> 添加 click 事件。我们不能确保 click 事件和 href 跳转的执行顺序。

示例

我们在 <list> 组件示例中开发了一个长列表页面展示 Github 上 star 数最高的 JavaScript 项目。在这里，我们继续丰富这个页面，为 Repo name 增加超链接，可跳转到对应的仓库去。

原始代码在这里：dotwe.org

第1步：添加 <a> 组件

我们将 Repo name 包裹在 <a> 组件和 <text> 组件内： <text class="item">Repo name: </text><a href="{{item.repo_url}}"><text>{{item.full_name}}</text></a>

<template>
  <div>
    <list class="list">
      <header class="header">
        <text class="title">Search Results</text>
      </header>
      <refresh style="width: 750; padding: 30;" onrefresh="refreshData">
        <text class="text"> ↓ Pull to refresh </text>
        <loading-indicator class="indicator"></loading-indicator>
      </refresh>
      <cell class="row" repeat="item in items" id="item-{{$index}}">
        <div>
          <text class="item">Repo name: </text><a href="{{item.repo_url}}"><text>{{item.full_name}}</text></a>
        </div>
        <div>
          <text class="item">Repo star: {{item.stargazers_count}}</text>
        </div>
      </cell>
      <loading onloading="loadingData" style="width: 750; padding: 30;" display="{{loadingDisplay}}">
        <text class="text">{{loadingText}}</text>
      </loading>
    </list>
    <div class="up" onclick="goToTop">
      <img class="img" src="https://img.alicdn.com/tps/TB1ZVOEOpXXXXcQaXXXXXXXXXXX-200-200.png"></img>
    </div>
  </div>
</template>

第2步：添加样式

因为 Weex 没有行内元素，这时我们发现 Repo name 被分为两行，我们可以通过 flex-direction: row; 将两个 <text> 排列在同一行。同时，我们为超链接增加样式。

<div style="flex-direction: row;">
  <text class="item">Repo name: </text><a href="{{item.html_url}}"><text class="link">{{item.full_name}}</text></a>
</div>

.link {
  color: #008cff;
  text-decoration: underline;
}

第3步：显示 web 页面

本来到这里，我们开发就已经完成了。但是 Playground App 不支持直接显示 H5 页面，因此这里我们需要使用 <web> 组件开发一个页面来显示 H5 页面。我们将我们实际要显示的 H5 页面 url 通过 weburl 这个 url 参数传递给这个 Weex 页面。比如这样： http://myweex.com/renderweb.js?weburl=https%3a%2f%2fm.taobao.com，http://myweex.com/renderweb.js 是 <web> 组件开发的页面的 JSBundle url，https%3a%2f%2fm.taobao.com 是被编码后的实际要显示的 H5 页面 Url。

这个组件开发很简单，我们将一个 <web> 组件全屏显示，然后通过 this.$getConfig() 方法获取到当前 JSBundle url，再从JSBundle url 中取 weburl 参数获取要显示的 H5 页面地址。需要注意的是，url 参数需要被编码，然后在此处被解码，以免 url 解析出错。

体验一下

<template>
  <div class="full-screen">
    <web class="full-screen" src="{{url}}"></web>
  </div>
</template>

<style>
.full-screen {
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
}
</style>

<script>
  module.exports = {
    data: {
      url: 'http://m.taobao.com'
    },
    created: function () {
      var bundleUrl = this.$getConfig().bundleUrl || '',
          bundleUrlParams = this.getUrlParams(bundleUrl)
          webUrl = bundleUrlParams.weburl || 'https%3a%2f%2fm.taobao.com'

      this.url = decodeURIComponent(webUrl)
    },
    methods: {
      getUrlParams: function (url) {
        var queryStr = url.split('?')[1] || '',
            queryArr = queryStr.split('&')
        var ret = {}

        queryArr.forEach(function (item, index) {
          var kv = item.split('='),
              key = kv[0]
              value = kv[1]

          if (key) {
            ret[key] = value
          }
        })

        return ret
      }
    }
  }
</script>

我们将这个代码上传，得到这个 bundle 的 URL 为：http://dotwe.org/raw/dist/f1fa0895d0fa0fd80896e02a589443dd.js。我们后面会用到这个地址。

第4步：处理 Url

接下来我们回到 <list> 页面代码中。我们拿到的 Repo 地址是未编码的 url，因此我们要对这个 url 进行编码，然后传递给 web 页面。

在全局添加常量 PAGE_URL 为显示 web 页面的 JSBundle url：

var PAGE_URL = 'http://dotwe.org/raw/dist/f1fa0895d0fa0fd80896e02a589443dd.js'

我们增加一个 processUrl() 方法并在获取到数据后调用：

processUrl: function (url) {
  if (url) {
    var encodeUrl = encodeURIComponent(url)

    return PAGE_URL + '?weburl=' + encodeUrl
  }
}

如此，这个长列表就开发完成了。
